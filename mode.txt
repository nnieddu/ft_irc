`O` et `o` pour tous ce qui demande a etre moderateur pour etre utiliser

`v` dois etre set pour pouvoir envoyer un message dans un channel qui aurrait
	le flag `m` de mis

`a` en mode utilisateur, est utilez quand on lui envoie un PRIVMSG, renvoie
	a l'expediteur le message set par l'utilisateur afk (set par la commande AWAY)

	en mode channel, permet au utilisateur d'etre anonyme en metant comme
	nickname anonymous, username anonymous et host anonymous au message.
	le server ne dois pas renvoyer de QUIT message pour les utilisateur quitant
	ce type de channel mais celui de PART a la place
	(a voir si vous voulez le faire)
	les retours de WHO WHOIS et NAMES ne doivent pas reveler la presence des autres
	utilisateurs sur les channels avec ce flag

`i`	en mode utilisateur fait que la commande NAMES ou WHO ne le fait pas apparaitre

	en mode channel, fait que l'on ne peut pas JOIN le channel, il faut utiliser
	la commande INVITE pour faire venir un personne sur le channel

`m`	mode channel, seul les operateurs du channel ou les membres possedant
	le flag `v` peuvent envoyer des message via PRIVMSG sur un channel

`n`	mode channel, les personnes exterieur au channel ne peuvent pas envoyer de
	messages sur le channel qui a ce flag de mis

`q`	pas compris

`p` mode channel, channel is ommited from command WHOIS

`s` obsolete donc pas a faire

`r` sert a op un utilisateur au hasard si il n'y a plus de channel operateur sur
	un channel au bout d'un certain temp (a faire?)

`t` mode channel, commande TOPIC seul les operateurs peuvent changer les topic

`k` mode channel, set un mots de passe au channel, JOIN demande un second parametre qui est
	le mots de passe du channel

`l`	mode channel, si set, limite le nombre d'utilisateurs sur un channel,
	une fois la limite atteinte, JOIN ne peut plus permetre les utilisateur de JOIN

`b`, `e`, `I` pas gerer pour cause d'usage de wildcard
